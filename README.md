
# Ejercicio QA – Eureka W1

Este ejercicio parcial tiene como objetivo medir el avance de la **Semana 1** del programa. Consiste en un sitio estático desarrollado con **HTML5** y **CSS**, distribuido en varias páginas temáticas.

## 🚀 Cómo ejecutar

1. Clona el repositorio:
   ```bash
   git clone https://github.com/Mouniee/Ejercicio-QA-Eureka-W1.git
   cd Ejercicio-QA-Eureka-W1
   ```
2. Abre `index.html` en tu navegador (doble clic o arrastrando el archivo).
   - Opcional: usa la extensión **Live Server** de VS Code para recarga automática.

## 📁 Estructura del proyecto

```
Ejercicio-QA-Eureka-W1/
├── index.html      # Página de inicio / menú principal
├── p1.html         # Sección 1
├── p2.html         # Sección 2
├── p3.html         # Sección 3
├── p4.html         # Sección 4
├── p5.html         # Sección 5
├── ref.html        # Referencias / bibliografía
└── style.css       # Estilos globales
```

## 🧭 Navegación

- Ingresa por `index.html` y utiliza los enlaces para acceder a `p1.html`–`p5.html`.
- `ref.html` incluye fuentes o material de apoyo (si aplica).

## ✅ Criterios sugeridos de evaluación (W1)

- Estructura **HTML5** válida (uso básico de etiquetas semánticas).
- Separación de estilos en `style.css`.
- Navegación funcional entre páginas (enlaces relativos).
- Contenido claro y consistente con los objetivos del ejercicio.
- Buenas prácticas: indentación, nombres descriptivos y organización.

## 🛠️ Requisitos

- Navegador moderno (Chrome, Firefox, Edge o Safari).
- No requiere instalaciones ni dependencias externas.

## 🔧 Recomendaciones de desarrollo

- Usa etiquetas semánticas: `header`, `nav`, `main`, `section`, `article`, `footer`.
- Aplica clases reutilizables en `style.css` y evita estilos inline.
- Valida HTML/CSS con los validadores de **W3C**.
- Antes de hacer *commit*, formatea el código y verifica que los enlaces funcionen.
